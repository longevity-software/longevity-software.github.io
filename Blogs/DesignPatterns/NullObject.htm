The Null Object Pattern

Picture the scene, you are writing an ADC library which allows users to request a number of samples from the underlying ADC, and they will provide a function to call when each sample is available. Or you have a state machine using a function pointer for the current state. In both these cases you may want to check that the function pointer is not Null before dereferencing it.
This is where the null object pattern come in, it allows a single Null check when the pointer is assigned, rather than wherever it is dereferenced, while providing the "do nothing" functionality which would be the intention of setting the function pointer to Null.
